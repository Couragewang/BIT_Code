第一个shell脚本

1. Linux环境下，新建以.sh结尾的文件，如 touch test.sh：

2. 打开文件.sh文件（如： vim test.sh），输入如下内容：

#!/bin/bash

echo "hello world!"

3. 保存退出！

4. 给shell脚本添加可执行权限，chmod u+s 你的.sh文件， 如：chmod u+s test.sh

5. 运行shell脚本， 如./test.sh

#! - shebang

1. 在shell脚本文件中，我们输入的第一行，一般按照'#!'开头，该符号我们称为”Shebang”或”Sha-bang”（还有其他名称，略）

2. 该符号指明了shell脚本的解释器程序，如/bin/bash

3. 如果脚本文件中没有#!这一行，那么脚本运行时，不会进行解释器替换。具体见《shell运行原理》部分

shell解释器

1. shell被称为Linux外壳程序，用来在交互模式下对用户的输入信息进行解析，shell也称为“命令行解释器”

2. 我们通常说shell代表我们的”命令行解释器“

3. 我们通常所说的shell脚本代表我们所编辑的文本文件

4. 注意区分，shell脚本， shell解释器，shell解释器进程，shell进程（后续会说明）

shell脚本运行方式

1. 假定我们的shell脚本名称叫做bit.sh

2. shell解释器进程会先fork一个子进程并调用exec执行./bit.sh这个脚本程序

3. 子进程程序替换shebang指定的解释器，用解释器程序的代码段替换当前进程,并且从解释器的_start开始执行,而这个文本文件bit.sh被当作命令行参数传给解释器

4. shell解释器对shell脚本中的命令逻辑进行逐行解释

shell特点

1. 非编译型语言

2. 若类型语言

3. 强大的批处理能力

4. 提高自动化部署，自动化运维效率

5. 编写简单，容易实现

常见shell版本

1. 文件/etc/shells给出了系统中所有已知(不一定已安装)的Shell,除了上面提到的Shell之外还有很多变种。
